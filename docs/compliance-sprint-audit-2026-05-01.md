## Implementation focus for this sprint

The highest-impact feasible repairs are to strengthen honesty and compliance without pretending true computer-vision parsing exists. The implementation should therefore make the fallback/review state explicit, fix deterministic requested-program and workstation math, add scenario demolition-rule metadata, improve layout hard-rule protection for fixed core elements, add intake controls for programming mode/style, and add regression tests proving the resulting scenario output remains source-of-truth aligned.
